Mapeamento Relacional Interativo

Modelo Abstrato de Persistência Física do Banco de Dados

Objetivo Estrutural do Sistema

  • Modelagem Relacional (3FN): Decomposição completa de tabelas multivaloradas para eliminação de redundâncias e anomalias de atualização.
  • Integridade no Kernel: Travas físicas do PostgreSQL (FOREIGN KEY, CHECK Constraints) blindando o armazenamento de dados direto no disco.
  • Aceleração de I/O: Índices estruturados em árvores balanceadas (B-Tree) mitigando buscas sequenciais exaustivas no disco rígido.

Segurança da Infraestrutura de Rede

Este ecossistema de dados opera sob rígido protocolo de segurança em trânsito, impedindo ataques de interceptação de dados.

Criptografia: TLSv1.3 (Transport Layer Security)

Algoritmo de Chave: RSA 2048-bit (Assinatura Digital)

Escopo: Criptografia fim-a-fim entre o Navegador do Recrutador e o Kernel do Back-end.

Mapeamento do Handshake Realizado:
Navegador ── ClientHello ──► Servidor
Navegador ◄── ServerHello + Cert ── Servidor
Navegador ── Troca de Chaves ──► Servidor
Sessão Segura ◄── [Canal Criptografado] ──► Dados Protegidos
